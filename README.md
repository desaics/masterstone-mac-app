# Masterstone CRM — Mac App

Native macOS desktop application for the Masterstone CRM, built with [Tauri 2](https://tauri.app/) (Rust shell + WebKit frontend).

## Status

| Phase | What it does |
|-------|--------------|
| **Session 1 (current)** | Scaffold + GitHub Actions build pipeline. Ships a "Hello World" `.app` to verify the toolchain end-to-end. |
| Session 2 | SQLite schema + JSON-backup importer. |
| Session 3 | Storage adapter — read paths (replaces `localStorage.getItem`). |
| Session 4 | Storage adapter — write paths + attachments. |
| Session 5 | OneDrive link rewrite + iPhone snapshot generator. |
| Session 6 | Hardening + production cutover. |

The HTML version of the CRM (`Masterstone_CRM.html`) continues to handle daily work throughout. The Mac app is built in parallel; cutover happens at the end of Session 6.

## Build

You don't build locally. Every push to `main` triggers GitHub Actions, which produces a universal macOS binary (Apple Silicon + Intel) and posts both the `.app` and `.dmg` to that run's artifacts page.

To download the latest build: **Actions** tab → click the most recent successful run → scroll to **Artifacts** at the bottom → download `Masterstone-app` (or `Masterstone-dmg`).

## First launch on macOS

macOS Gatekeeper will warn that the app isn't notarized. This is normal — the build is unsigned by design (decision 3A in the architecture doc). Bypass: **right-click the `.app` → Open**. macOS remembers per-build and won't ask again.
